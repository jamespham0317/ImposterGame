Definitions for all JSON messages sent between Frontend and Backend and fields they include

Each JSON message starts with a 'type' field defining the purpose of the request/message
For example, a request to create a room would have 'type' : 'create-room' 

Frontend -> Backend:
    Request: create-room
        playerId
    Response: room-created
        roomId
        playerId

    Request: join-room
        roomId
        playerId
    Response: room-joined
        roomId
        playerId
        playerList

    Request: start-game
        roomId
    Response: game-started
        playerList
        imposterId
        problem
        testCycle

    Request: run-test-cycle
        roomId
        playerId
        code 
    Response: test-results # if not all tests passed
        outputList
        passedList
    Response: start-vote # if all tests passed
        commits
        votes

    Request: next-turn
        roomId
        playerId
        code
    Response: next-turn
        currentPlayer
        code

    Request: cast-vote
        roomId
        playerId
    Response: vote-casted
        votes

Backend -> Frontend
    Message: player-joined
        playerList

    Message: time-left
        timeLeft

    Message: turn-over

    Message: vote-over
        votedList
        votedCorrectly
