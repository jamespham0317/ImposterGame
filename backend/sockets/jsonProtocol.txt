Definitions for all JSON messages sent between Frontend and Backend and fields they include

Each JSON message starts with a 'type' field defining the purpose of the request/message
For example, a request to create a room would have 'type' : 'create-room' 

Frontend -> Backend:
    Request: create-room
        playerId
    Response: room-created
        roomId

    Request: join-room
        roomId
        playerId
    Response: room-joined
        roomId
        playerList

    Request: start-game
        roomId
    Response: game-started
        playerList
        imposterId
        problemTitle
        problemDifficulty
        problemDescription
        problemExamples
        problemConstraints
        problemTopics
        problemCode
        TestInputList
        TestExpectedOutputList

    Request: run-tests
        roomId
        code 
    Response: test-results # if not all tests passed
        testPassesList
        OutputList
    Response: start-vote # if all tests passed
        commits

    Request: cast-vote
        roomId
        playerId
    Response:
        voteList

Backend -> Frontend
    Message: time
        time

    Request: turn-over
        currentPlayer
    Response: 
        code

    Message: vote-over
        voted
        votedCorrectly
